Welcome to BoilerplateJS!

Congratulations! You are one step closer to creating your next big Javascript application! This demo shows you some of the common user scenarios that you can implement with BoilerplateJS. Checkout the product modules by navigating through the left navigation pane.

Product Module A

The panels under this section show the usage of KnockoutJS with BoilerplateJS.

Click Counter is a simple adaptation of knockoutJS’s Click Counter example.
Knockout MVVM shows a more complex usage of knockoutJS’s MVVM architecture and its data bindings. The data for this panel is loaded from the server.

Product Module B

The panels under this section show the usage some of the advanced patterns.

Complex Dashboard uses a shared view model. There are two views in this dashboard that use the same shared view model, which enables them to respond to changes among the views.
The List-Detail panels show a list view and a detail view together with URL based navigation.

Languages and Themes

Languages and Themes are two feature plugins that we developed for BoilerplateJS.

We realized that for some applications, multilingual support is essential. On this demo we use RequireJS’s i18n plugin to manage two language packs.

Presentation and styling should be separate from the rest of the logic of your code. Careful thought went into designing BoilerplateJS’s style sheet structure which allows us to change between themes seamlessly. Try changing into different themes from the top combo box.

We strongly believe loose coupling is the key to maintainable and adaptable code, and that’s why we do not want to tie you up with a specific technology or library. To prove this point, we’ve implemented the language and themes modules using pure jQuery, and no Knockout bindings!
